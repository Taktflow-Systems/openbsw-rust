## Initial observations

Before the first-slice changes, the portable workspace passed 1,062 listed
Rust test entries and `--all-features` checks. Three intrusive-list doctests
were intentionally ignored. The F4 cross-check emitted 27 diagnostic warning
records and the G4 cross-check emitted 32, principally inactive-target MMIO
items and references to mutable statics. The diagnostic line printed by Cargo
made the raw console headline one higher for each target.

The HIL suite collected 1,086 cases but had one collection error:
`hil/test_multiframe_diag.py` imports a missing `helpers.isotp.is_cf`. Hardware
execution was not attempted. This is baseline debt and is not counted as
passing evidence.

Do not set a bare `RUSTFLAGS` while producing release size evidence: Cargo
environment flags replace the target linker flags in `.cargo/config.toml`.
The feature scripts use `RUSTFLAGS=-D warnings` only with `cargo check`, which
does not link an image.

`cargo fmt --all -- --check` currently reports pre-existing formatting drift
across legacy examples and modules. The files changed by this slice are
formatted, but a repository-wide mechanical rewrite is deliberately deferred
to a separately reviewed package.

The reference smoke intentionally ends via a five-second timeout. In the
unprovisioned host environment its UDP demonstration sends report a network
error; that does not prevent lifecycle startup, but it is not evidence of live
Ethernet or DoIP parity.

## Known functional gaps confirmed by inspection

- `bsw-com` supports byte-aligned signals only.
- `DiagCanTransport` has a 256-byte fixed diagnostic payload ceiling.
- UDS routine-control uptime and stop/result paths contain placeholder behavior.
- Protocol and MMIO logic is duplicated across several BSP examples.
- No mandatory parity row has sufficient evidence to be marked complete yet;
  see the generated `status.md`.
